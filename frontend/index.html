Design Guide
How this interface is built — written for a builder who wants to replicate it
The premise

One ambient light field sits fixed behind everything. Three tiers of frosted glass float above it. The cyan #A7EBF2 is reserved for signal — live data, the active state, focus — never decoration. Everything else stays quiet so the numbers can speak.

The glass recipe

Real glassmorphism needs three things: something behind it worth blurring, a tinted translucent fill, and an edge that catches light. The ambient field supplies the first; every card gets the other two:

/* tier 1 — cards */
.card {
  background: linear-gradient(150deg,
    rgba(255,255,255,0.055) 0%,
    rgba(255,255,255,0.015) 42%,
    rgba(167,235,242,0.028) 100%);
  backdrop-filter: blur(16px) saturate(150%);
  border: 1px solid rgba(167,235,242,0.09);
  box-shadow:
    inset 0 1px 0 rgba(255,255,255,0.06),  /* light catch */
    0 12px 36px -14px rgba(0,0,0,0.55);   /* depth */
}

The diagonal tint gradient matters more than the blur — it fakes a light source at the top-left, which is what makes flat rectangles read as panes. The saturate(150%) pushes the ambient colour back through the frost.

  • Depth tiers: cards blur 16px, the nav rail and header blur 24px, modals blur 32px over a dimmed scrim. Deeper in the z-stack = heavier frost.
  • Blur budget: one backdrop-filter per top-level surface, never nested. Rows inside a card use plain translucency (rgba(255,255,255,0.035)). Stacked blur is a GPU tax with no visual payoff.
  • The ambient field is two fixed pseudo-elements: layered radial gradients (cyan 7%, blue 8%, violet 5%) drifting on a 70s transform loop, plus an SVG feTurbulence noise tile at 3% opacity to kill gradient banding.
Colour system

Near-black base, one signal accent, three semantic states, four ink tiers. Accent usage is the discipline: if cyan appears somewhere, it must mean something is live, active, or focused.

--bg#0A0B0Dbase field
--glow#A7EBF2signal only — data, active, focus
--interactive#1A8FBFbutton fills, user bubbles
--success#3EDCB4done, cleared, profit
--danger#FF5C5Clocked, drift, loss
--warn#F5C842behind schedule
--purple#A78BFAacademics domain

Ink comes in four alphas of the same near-white: --text 100%, --text-2 64%, --muted 52%, --faint 32%. The 52% floor keeps body copy at ≥4.5:1 contrast on the base; 32% is allowed only on decorative micro-labels. Hairlines are cyan at 5–22% alpha rather than grey — it keeps the whole surface in one colour temperature.

Type system
$4,280 / 48.02s
Bricolage Grotesque carries the hero numerals. Geist carries sentences like this one.
GEIST · CARD TITLES · 09:41 · +1.5R
  • Bricolage Grotesque (display). Ink-trap grotesque with characterful numerals — chosen over the saturated AI-default cluster (Inter/Space Grotesk/Syne everywhere). The ink traps read as crafted against the cold glass. Weight 700, tracking −0.03em, only at hero and page-title sizes.
  • Geist (body + technical voice). One face, two registers. Sentences run 11.5–13px in normal case. The technical register — card titles, timestamps, badges, table cells — runs uppercase at 9.5–10.5px with 0.14–0.22em tracking and true tabular figures via font-variant-numeric: tabular-nums, so columns of R-multiples and times never wobble. The machine voice comes from case, tracking and size, not from a terminal typeface.
  • The pairing axis: one characterful display, one precise instrument face doing body and data. Two families total keeps the glass surface quiet; literal code samples fall back to the system monospace (--font-code) and load nothing.
Motion

Everything runs on five curves and three durations. The two already in the codebase were kept as the physical identity:

--ease-ios:    cubic-bezier(0.32, 0.72, 0, 1);   /* drawers, page slide */
--ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1); /* checkmark pop */
--ease-out:    cubic-bezier(0.23, 1, 0.32, 1);    /* entrances, presses */
--t-fast: 120ms;  /* press feedback */
--t-base: 190ms;  /* hover, toggles */
--t-slow: 300ms;  /* modals, rail */
  • Entrances draw once. Bars, staggers and the radar animate under a .page.fade-up scope that is removed ~700ms after navigation — so the 30-second background sync re-renders without replaying motion. Data updates should never look like page loads.
  • The radar choreographs itself: rings fade in staggered (0–150ms), the data polygon scales up from 55% (180ms, ease-out-quart), six dots pop in sequence (340–540ms, spring), labels last. Total under 900ms, runs on arrival of fresh scores.
  • Press is physical: every pressable scales to 0.97 at 120ms. Hover lifts 1px and brightens the border — gated behind @media (hover:hover) and (pointer:fine) so touch devices never get sticky hover states.
  • Exits are faster than enters (toasts leave in 180ms, arrive in 300ms), and prefers-reduced-motion collapses all movement to instant fades — the ambient drift stops entirely.
  • Only compositor properties animate — transform and opacity. The one exception is the nav rail's width, which was made an overlay precisely so its expansion never reflows page content.
Deliberately avoided, and why
  • Glass on everything. Blur is the material of top-level surfaces only. When every nested row frosts, nothing reads as elevated and the GPU pays for it.
  • Side-stripe borders (border-left: 3px solid accent) on cards, alerts, nav items. Replaced with full tinted borders, background washes, or a lit dot. Stripes are the laziest possible hierarchy signal.
  • Gradient text. Emphasis here is weight, size, or the one accent colour — never background-clip: text.
  • Sci-fi HUD props. No boot sequences, no scanlines, no particle fields, no corner brackets. The instrument feel comes from mono numerals and restraint, not props.
  • Emoji as icons. The nav uses a hand-drawn 16px stroke set (1.5px, round caps) so weight and rhythm stay uniform across platforms.
  • scale(0) entrances and transition: all. Nothing real appears from nothing; entrances start ≥ scale(0.55) with opacity. Transitions name their properties.
  • Hover-only affordances. Row delete buttons that reveal on hover also surface at 45% opacity on coarse pointers.
The constraint that shaped it

This was a skin-only rebuild: every fetch, endpoint, state shape and business rule survived byte-identical. All motion is CSS-first (keyframes + transitions on injected markup), which is why entrance animation is class-scoped rather than JS-orchestrated — the render functions re-run freely and the presentation layer decides when movement is meaningful.
